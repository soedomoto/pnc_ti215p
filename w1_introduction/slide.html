What is Database?

Organized collection of data or information
Different data may need different type of database
Choose right database based on how you will organize, store, and access data

Types of Databases

The Idea: How structure the information
Generally divided into two:
Relational Database
Non-Relational Database

Relational Database

Multiple tables
Can be joined
Most common. Widely used
Example: MySQL, PostgreSQL, SQLServer, Oracle

Non-Relational Database (NoSQL)

No Table, No Structure --> Collections
Not support join
Alternative. Depend on use case
Example: MongoDB, Redis

NoSQL: Hierarchical Database

A single object (parent) has one or more objects beneath it (child)
Example: Windows registry

NoSQL: Document databases

JSON, BSON, XML
Example: MongoDB

NoSQL: Key-value stores

Most simple format
Example: Redis

NoSQL: Column-oriented databases

Relational DB -> row based
Column-oriented DB -> column based
Commonly used for analytics
Example: Cassandra

Relational Database Structure

Table: collection of entities
In OOP terminology --> collection of similar objects
Row: A single entity or observation
In OOP terminology --> a single object
Column: characteristics of entity
In OOP terminology --> field or attribute

String --> varchar, nvarchar, text
Decimal --> decimal, float, numeric
