SQL: Motivation

What is in Relational Database?
Structure/Schema and Data
Structure/Schema might be: Created, Altered, Dropped
Data might be: Selected, Inserted, Updated, Deleted
Access might be: Granted and Revoked

SQL

SQL stands for Structured Query Language
SQL lets you access and manipulate databases
SQL became ANSI in 1986, and ISO 9075 in 1987
Beware of Non-Compatible SQL between vendors

SQL: Motivation

What kind of SQL to?
Create, Alter, Drop database Structure/Schema: DDL
Select, Insert, Update, Delete Data: DML
Grant and Revoke Access: DCL

DDL: Create

Create Database
              
                CREATE DATABASE database_name
Create Table
              
                CREATE TABLE table_name (
                  field_1 TYPE(length) [NOT NULL] [DEFAULT value],
                  field_2 TYPE(length) [NOT NULL] [DEFAULT value],
                  ...,
                  PRIMARY KEY (field_1)
                )

DDL: Create

Create Index
              
                CREATE INDEX index_name
                ON table_name (field_1, field_2, ...);
Create Unique Index
              
                CREATE UNIQUE INDEX index_name
                ON table_name (field_1, field_2, ...);

DDL: Alter

Alter Table: Add Column
              
                ALTER TABLE table_name
                ADD column_name TYPE(length) [NOT NULL] [DEFAULT value];
Alter Table: Drop Column
              
                ALTER TABLE table_name
                DROP column_name;
Alter Table: Modify Column
              
                ALTER TABLE table_name
                MODIFY column_name TYPE(length) [NOT NULL] [DEFAULT value];

DML: Drop

Drop Database
              
                DROP DATABASE database_name
Drop Table
              
                DROP TABLE table_name
Drop Index
Note: Drop Index is different between vendors
              
                DROP INDEX index_name

DML: Insert

INSERT SINGLE
              
                INSERT INTO table_name (field_1, field_2, ...)
                VALUES (value_1, value_2)
INSERT MULTIPLE
              
                INSERT INTO table_name (field_1, field_2, ...)
                VALUES (value_1, value_2), (value_1, value_2), ...

DML: Update

              
                UPDATE table_name SET 
                  field_1 = new_value,
                  field_2 = new_value
                WHERE field_key = condition

DML: Delete

              
                DELETE table_name 
                WHERE field_key = condition
